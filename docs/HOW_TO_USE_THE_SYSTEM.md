## 1. Getting Started

### Logging in

1. Open the system in your browser (e.g. `https://yoursite.com` or `https://yoursite.com/en/auth/login`).
2. Enter your **Username** and **Password**.
3. Optionally check **Remember me** to stay logged in.
4. Click **Login**.

After login you are taken to the **Dashboard**. If you do not have an account, ask an administrator to create one.

### Logging out

Use the logout option in the top-right (user menu) to sign out.

### Language

If the site supports multiple languages, use the language selector (e.g. in the header or footer) to switch (e.g. English / Swahili).

## 2. Dashboard and Navigation

### Dashboard

- The **Dashboard** is the home page after login.
- It shows an overview of the cooperative (widgets/summaries depend on your role and configuration).
- Use the sidebar to open other sections.

### Sidebar menu

- **Dashboards** – Return to the main dashboard.
- **Loan Calculator** – Calculate loan amounts and schedules (see [Loan](#8-loan)).
- Other items (Members, Mortuary, Contribution, Saving, Share, Loan, Finance, Reports, Messaging, Settings, User Manager, Data Migration) appear according to your **permissions**.

Only menu items you are allowed to use are visible. If you do not see a section, you do not have access to it.

## 3. Members

Use **Members** to manage cooperative members and groups.

| Menu item | Purpose |
|-----------|--------|
| **Member Registration** | Register a new member (if you have “Register new member” permission). |
| **Non-Member Registration** | Register a non-member (if allowed). |
| **Member List** | View and search the list of members. |
| **Add Group** | Create or manage member groups (if allowed). |
| **Member Group List** | View member groups. |
| **Member Current State** | View current status/summary of members. |

**Typical workflow**

1. Register members via **Member Registration** (or **Non-Member Registration**).
2. Organize them using **Add Group** / **Member Group List**.
3. Use **Member List** to search and view details.
4. Use **Member Current State** for an overview of member status.

| Menu item | Purpose |
|-----------|--------|
| **Mortuary Master List** | View mortuary master/configuration list. |

## 8. Loan

The **Loan** module handles loan applications, approval, disbursement, and repayment.

### Loan Calculator

- Use **Loan Calculator** from the sidebar to estimate loan amount, term, and installments before applying.

### Loan menu (staff/admin)

| Menu item | Purpose |
|-----------|--------|
| **Loan View List** | View all loans and their status. |
| **Loan Application** | Enter new loan applications. |
| **Loan Evaluation** | Evaluate applications (if you have evaluate permission). |
| **Loan Approval** | Approve or reject loans (if you have approve permission). |
| **Loan Disbursement** | Disburse approved loans. |
| **Loan Repayment** | Record loan repayments. |
| **Loan Beginning Balance** | Manage opening loan balances (if enabled). |

**Typical workflow**

1. Member or staff submits a **Loan Application**.
2. Staff **Evaluate** the application.
3. Authorized user **Approve** or reject.
4. After approval, **Disburse** the loan.
5. Record **Repayments** in **Loan Repayment**.
6. Use **Loan View List** and **Reports** for lists, statements, and aging.

| Menu item | Purpose |
|-----------|--------|
| **Finance Account List** | Chart of accounts. |
| **Chart Type List** | Account chart types. |
| **Chart Sub Type List** | Account chart sub-types. |
| **Beginning Balance List** | Opening balances for accounts. |
| **Cash Receipt List** | Record and view cash receipts. |
| **Cash Disbursement List** | Record and view cash disbursements. |
| **Customer** | Manage customers. |
| **Sales Quote / Sales Invoice** | Create quotes and invoices for customers. |
| **Supplier List** | Manage suppliers. |
| **Purchase Order / Purchase Invoice** | Create POs and purchase invoices. |

4. Use **Reports** (e.g. Trial Balance, Income Statement, Balance Sheet, Cash Flow) for period-end and reporting.

For more detail on Cash Receipt and Cash Disbursement, see `docs/README_FINANCE_MODULES.md`.

## 10. Reports

**Reports** is a central place for all report types.

- From the sidebar, click **Reports** to open the report home page.
- The page lists report categories (e.g. Finance, Member, Mortuary, Contribution, Loan, Saving, Share).
- Click a report name to open it; then set date range, member, or other filters as shown on screen.
- Use **Print** or **Export** (when available) to print or download.

**Examples of report categories**

- **Finance:** General Ledger, Trial Balance, Income Statement, Balance Sheet, Cash Flow, Journal reports.
- **Member:** Member list, Member profile, Registration fee collection.
- **Mortuary:** Mortuary fund balance, member mortuary statement, transactions, summary.
- **Contribution (CBU):** CBU balance, member CBU statement, transactions, summary.
- **Loan:** Loan list, loan balance, repayment schedule, loan statement, aging, processing fee collection, etc.
- **Saving:** Saving account list, statement, transactions, summary.
- **Share:** Share balance, member share statement, transactions, summary.

Your visible reports depend on your permissions and module access.

## 13. User Manager

The **User Manager** is for administrators to manage system users and roles.

| Menu item | Purpose |
|-----------|--------|
| **Create Group** | Create a new user group (role). |
| **Group List** | View and edit user groups. |
| **Create User** | Add a new system user (username, password, group). |
| **User List** | View, edit, activate/deactivate, or delete users. |

**Managing users**

1. **User List** – Search by name, email, or username; filter by group; sort by date created, name, or email. Columns include Date Created, First Name, Last Name, Username, Email, Groups, Status, and Action (Edit/Delete).
2. **Create User** – Fill in first name, last name, email, phone, username, password, and assign a group. Save.
3. **Edit user** – From User List, click Edit; update details or password as needed.
4. **Activate/Deactivate** – Use the status link in User List to activate or deactivate an account.
5. **Delete** – Use Delete in User List (soft delete; user is hidden from the list).

**Managing permissions**

1. Open **Group List**.
2. Click the group you want (e.g. to set permissions).
3. Use the **Privilege / Role** (or similar) screen to enable or disable permissions per module (Members, Mortuary, Contribution, Saving, Share, Loan, Finance, Reports, Messaging, Settings, User Manager).
4. Save. Users in that group will have the updated access.

## 14. Data Migration

**Data Migration** is used to import data from files (e.g. Excel/CSV) into the system.

| Menu item | Purpose |
|-----------|--------|
| **Import Member** | Import members from a file. |
| **Import Member Mortuary** | Import mortuary data. |
| **Import Member Contributions** | Import contribution data. |
| **Import Member Share** | Import share data. |
| **Import Loan Information** | Import loan records. |
| **Import Loan Repayment Schedule** | Import repayment schedules. |
| **Import Loan Repayment Transactions** | Import repayment transactions. |

**General steps**

1. Prepare your file in the format required (check on-screen instructions or sample templates).
2. Open the relevant **Import** page.
3. Upload the file and map columns if asked.
4. Run the import and correct any errors reported.

Use with care; wrong data can affect reports and balances. Prefer doing imports in a test environment or after a backup.

## 15. Member Portal (for Members group)

If you belong to the **Members** user group, your menu is different: you see a simplified “member” menu.

- **Loan** – View your loans, apply for a new loan, and respond to **Loan guarantor requests**.
- **Profile** – View your member profile.
- **Member Contribution** – View your contribution (CBU) report.
- **Member Shares** – View your share report.
- **Loan Repayment Schedule** – View your repayment schedule.
- **Loan Statement** – View your loan statement.

You do not see internal admin menus (e.g. User Manager, Settings, full Finance).

## 16. Change Password

All users can change their own password.

- Click **Change Password** in the sidebar (or open it from the user menu in the header).
- Enter your **Current password**, then **New password** and **Confirm new password**.
- Save. You may be asked to log in again with the new password.

## Getting help

- **Permissions:** If you cannot see a menu or action, your user group may not have the required permission. Ask an administrator to check **User Manager → Group List → [your group] → Privilege**.
- **Errors:** Note the exact message and the page/action you were doing; report to your system administrator or IT support.
- **Finance modules:** See `docs/README_FINANCE_MODULES.md` for Cash Receipt and Cash Disbursement details.
